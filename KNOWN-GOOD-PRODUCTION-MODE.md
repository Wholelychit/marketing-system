# Known Good Production Mode

## Source

Gerry reported one working ChatGPT/Codex screen that ran all day with no stops and no file problems.

Use that behavior as the target workflow for all website production.

## Required behavior

- Keep working through safe tasks.
- Do not ask Gerry to create files.
- Do not ask Gerry to paste files.
- Do not ask Gerry to update files.
- Do not ask for routine thumbs-up approval.
- Use direct repository file updates.
- Use full-file replacement when cleaner than small patches.
- Commit useful changes.
- Report commits and blockers only.

## Working production pattern

1. Read repo instructions.
2. Review current files.
3. Fix safe issues directly.
4. Commit useful changes.
5. Continue to the next safe task.
6. Report after a useful batch.

## Stop only for real blockers

Stop only for:

- blocked write
- major code deletion
- framework change
- private key or credential
- payment setup
- live tracking setup
- live ads
- unclear repo state

## Bad production pattern

Do not do these:

- ask Gerry to paste code
- ask Gerry to create files
- ask Gerry to update files
- keep repeating plans
- stop after one small fix when safe next work is clear
- switch a simple site to a framework
