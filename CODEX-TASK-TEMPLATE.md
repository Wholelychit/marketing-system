# Codex Task Template

Last updated: 2026-05-28

This is the reusable starting prompt for safe Codex work in Wholelychit repositories.

## Starter Prompt

Review this repository and continue the safe production build queue.

Read these files first when they exist:

- `README.md`
- `AGENTS.md`
- `AGENT-INSTRUCTIONS.md`
- `LOCKED-CHECKPOINT.md`
- `FILE-MANAGEMENT.md`
- `PROJECT-STATUS.md`
- `CODEX-WORKFLOW.md`

## Workflow

Codex should handle routine repo files directly. Gerry should not have to manually paste or rebuild files when Codex has write access.

Keep the current tech stack. Use the existing project structure. Make safe edits, commit useful changes, and continue through the safe queue until there is a real blocker.

## Safe Build Queue

1. Check homepage title, meta description, canonical URL, footer, and navigation.
2. Add or update `README.md`.
3. Add or update `AGENTS.md`.
4. Add or update `AGENT-INSTRUCTIONS.md`.
5. Add or update `PROJECT-STATUS.md`.
6. Add or update `CODEX-WORKFLOW.md`.
7. Add or update `robots.txt`.
8. Add or update `sitemap.xml`.
9. Add or update `SEO-CHECKLIST.md`.
10. Add or update `LAUNCH-CHECKLIST.md`.
11. Add or update `REVIEW-CHECKLIST.md`.
12. Fix safe broken links, page copy issues, mobile layout issues, and accessibility issues.
13. Record blockers in the status file.

## Real Blockers

A real blocker means the work needs missing repo permission, account sign-in, credentials, billing, payment setup, live ads, live tracking, framework migration, major code deletion, or an unclear production risk.

## Good Commit Messages

- `Add Codex workflow documentation`
- `Update agent instructions for direct repo edits`
- `Add project status and safe queue`
- `Add SEO checklist`
- `Update launch checklist`
- `Fix navigation links`
- `Improve mobile homepage layout`

## Reporting

Report after several useful commits or when a real blocker appears.
