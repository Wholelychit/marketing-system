# Time Management Rules

## Purpose

Reduce wasted loops, repeated planning, and repeated file changes. Move projects forward in useful safe batches.

## Main rule

Do not spend the session repeating plans. Review files, fix safe issues, commit, and continue.

## Gerry time rule

Gerry should not be asked to paste, create, or update files manually.

Only ask Gerry for actions AI cannot do, such as sign-ins, approvals, account permissions, domain settings, billing decisions, or choosing between major business directions.

## Session flow

1. Identify the repo.
2. Read its control files.
3. Confirm the tech stack.
4. Pick the highest-value safe queue item.
5. Make the change directly.
6. Commit the change.
7. Continue to the next safe item.
8. Report only after useful progress.

## Useful batch size

A good batch usually includes one of these:

- 2 to 5 small file improvements
- 1 complete control-system upgrade
- 1 homepage/SEO/footer improvement
- 1 launch-readiness pass
- 1 blocker cleanup pass

## Stop points

Stop only for:

- blocked write access
- unclear repository state
- major code deletion risk
- framework change risk
- private keys or tokens
- payment setup
- live tracking setup
- live ad setup
- domain/account permission tasks

## Do not stop for

- missing planning files
- missing checklists
- minor copy issues
- weak SEO text
- missing README sections
- missing robots or sitemap files
- missing footer guidance
- missing prompt templates

Fix those directly.

## Anti-repeat checklist

Before repeating a task, check:

1. Was this already completed?
2. Is there a checkpoint file?
3. Is there a status file?
4. Is the same file being edited again for no new reason?
5. Would the next action be better than rewriting the same plan?

## Report timing

Report after useful work, not after every tiny action.

Report should include:

1. Files changed.
2. Commits made.
3. What improved.
4. Blockers.
5. Next safe queue.

## Default next action rule

When unsure, choose the next safe action that improves clarity, launch readiness, SEO readiness, user trust, or monetization readiness.
