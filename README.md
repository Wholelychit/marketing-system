Website code stays in each website repository. Marketing strategy and reusable systems live here.

## System Files

Use these files to run the marketing machine without repeating work:

- `PROJECT-STATUS.md` — current project priorities, blockers, and next queue.
- `CONTENT-ENGINE.md` — repeatable content production process.
- `MARKETING-CALENDAR.md` — weekly publishing schedule.
- `SEO-SYSTEM.md` — keyword, article, page, and internal-link process.
- `SOCIAL-POST-TEMPLATES.md` — reusable posts for YouTube, Instagram, Facebook, TikTok, LinkedIn, Pinterest, Reddit, and X.
- `CANVA-ASSET-SYSTEM.md` — image/video asset rules and reusable Canva asset plan.
- `REVIEW-CHECKLIST.md` — final review checklist before publishing.

## Workflow

1. Choose one project focus for the week.
2. Pick one traffic channel and one conversion goal.
3. Create helpful content from the templates in this repo.
4. Create matching visuals in Canva when needed.
5. Publish to the correct website/social channel.
6. Track what was published and what needs improvement.
7. Improve weekly.
